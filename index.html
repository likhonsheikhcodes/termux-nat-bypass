Termux NAT Bypass

Expose your local servers securely using Tor, Ngrok, and Tmate. Bypass NAT restrictions and gain remote access effortlessly.

Introduction

Termux NAT Bypass is a toolkit that allows you to expose your local services to the internet, even when you're behind NAT or firewalls. This tool uses powerful utilities like **Tor**, **Ngrok**, and **Tmate** to securely create remote tunnels, allowing you to access your services from anywhere.

Installation

Follow the steps below to set up the Termux NAT Bypass tools:

Step 1: Clone the Repository

git clone https://github.com/likhonsheikhcodes/termux-nat-bypass.git

Step 2: Install Dependencies

pkg update && pkg upgrade -y
pkg install tor openssh tmate proxychains-ng -y

Step 3: Set Up Tor

Configure Tor for Onion services by editing the `torrc` file:

nano $PREFIX/etc/tor/torrc

Add the following to expose SSH through Tor:


SOCKSPort 127.0.0.1:9050
HiddenServiceDir /data/data/com.termux/files/home/.tor/hidden_ssh
HiddenServicePort 22 127.0.0.1:8022

Create the directory for the hidden service:

mkdir -p ~/.tor/hidden_ssh

Start Tor:

tor

Your SSH service will be available at the `.onion` address stored in `~/.tor/hidden_ssh/hostname`.

Step 4: Install Ngrok

wget https://bin.equinox.io/c/4VmDzA7iaHb/ngrok-stable-linux-amd64.zip
unzip ngrok-stable-linux-amd64.zip
mv ngrok $PREFIX/bin/

Authenticate Ngrok:

ngrok authtoken YOUR_AUTH_TOKEN

Expose a local service via Ngrok:

ngrok tcp 8022

Step 5: Install Tmate

pkg install tmate

Start a session with:

tmate

Share the session link with collaborators to work together remotely.
